# 空闲磁盘空间管理：成组连接法（Unix 专用）

## 定义

**成组连接法**是 Unix System V 采用的空闲磁盘块管理方法，它结合了**空闲表**和**空闲链表**的优点，既支持高效的分配与回收，又控制元数据开销。

**核心思想**：将空闲盘块分组，每组用一个**超级块中的栈**或**分组索引块**来管理，组间通过指针连接。

**对比其他空闲管理方法**：

| 方法 | 分配速度 | 元数据开销 | 适用场景 |
|---|---|---|---|
| 空闲表法 | O(n) | 小 | 连续分配需求 |
| 空闲链表法 | O(1) | 大（每块均需指针） | 小型 FS |
| **成组连接法** | **O(1)** | **小（分组存储指针）** | **大/中型 FS** |
| 位示图法 | O(1~n) | 适中 | 通用 FS |

## 核心原理

### 数据结构

空闲盘块按组组织，每组含 $N$ 个空闲盘块号（Unix 中常见 $N=100$）。

**超级块（Super Block）** 中维护一个栈结构：

```
                   ┌─────────────┐
                   │   超级块      │
                   │  s_nfree=N   │  ← 栈顶指针（第一组空闲块数）
                   │  s_free[0]   │  ← 第1块号
                   │  s_free[1]   │  ← 第2块号
                   │     ...      │
                   │  s_free[N-1] │  ← 第N块号（作为下一组的索引块）
                   └──────┬──────┘
                          │
                   ┌──────▼──────┐
                   │ 第2组索引块    │
                   │  空闲块数=N-1 │  ← 最后一块指向下一组
                   │  块号1        │
                   │  块号2        │
                   │     ...      │
                   │  块号N-1 → 指向第3组索引块
                   └─────────────┘
```

**关键规则**：
- 最后一组的最后一个空闲块号填 `0`（或 `null`），表示链尾
- 每组空闲块的最后一个块**不空闲**——它存的是下一组的索引信息，自身被当作索引块占用

### 分配过程

1. 检查超级块中 `s_nfree`：
   - 若 `s_nfree > 1`：直接从栈顶弹出一个块号（`s_free[--s_nfree]`）
   - 若 `s_nfree == 1`：弹出最后一个块号，该块不是数据块，而是**下一组的索引块**。需要将其内容读入超级块栈，然后继续分配
   - 若 `s_nfree == 0`：表示无空闲块

**时间复杂度**：$O(1)$（仅弹栈操作）

### 回收过程

1. 检查超级块中 `s_nfree`：
   - 若 `s_nfree < N`：直接压栈（`s_free[s_nfree++] = 块号`）
   - 若 `s_nfree == N`：表示超级块栈已满，先将当前超级块栈内容写入待回收的块（使其成为新的索引块），然后重置栈，把该块压入栈顶

**时间复杂度**：$O(1)$

### 分配/回收示例

**初始状态**：超级块栈有 100 块（$N=100$）

**分配 1 块**：
```
s_nfree: 100 → 99
返回：s_free[99]
```

**分配 98 块后**：
```
s_nfree: 2
超级块栈：s_free[0] = 块号X, s_free[1] = 块号Y（Y是下一组索引块）
```

**再分配 1 块**（关键步骤）：
```
s_nfree: 2 → 1
返回：s_free[1] = 块号Y（但块号Y实际不是数据块！）
```

此时 `s_nfree == 1`，意味着弹出的最后一块是**下一组的索引块**。OS 需要：
1. 读出块号 Y 的内容（其中存储了下一组 N-1 个空闲块号）
2. 将读出的内容填入超级块栈
3. 设置 `s_nfree = N-1`
4. 然后从新栈顶再分配一个给请求者

## 核心定理 / 性质

### 成组连接法 vs 位示图法

| 特性 | 成组连接法 | 位示图法 |
|---|---|---|
| 内存占用 | 仅超级块中的一个栈 | 需要整个位图（$总块数/8$ 字节） |
| 分配速度 | $O(1)$ | $O(1)$（配合偏移计算） |
| 碎片 | 无额外碎片 | 无额外碎片 |
| 实现复杂度 | 较高（需要处理组切换） | 较低 |
| 批量分配 | 自然支持（连续弹栈） | 需要扫描 |
| 系统崩溃恢复 | 较难（超级块中的栈是关键） | 较容易（位图可扫描重建） |

### 与其他文件系统概念的联动

- **FCB 中的起始簇号**：分配到的块号最终填入 FCB 的起始位置字段
- **超级块**：成组连接法的栈存储在超级块中，超级块本身通常放在磁盘固定扇区（如第 2 扇区）
- **与 FAT 对比**：FAT 用簇号链表记录已分配空间，而成组连接法管理的是空闲空间

## 计算方法 / 技巧

### 1. 超级块栈大小与磁盘分区的关系

设：
- 每盘块大小 $B$（通常 512 B 或 1 KB）
- 每盘块可存的块号数 = $B / 4$（每个块号 4 B）
- Unix 中 $N$ 通常取 100（不超过一块中能容纳的块号数）

对于 $B=1024\text{ B}$，一块可存 $1024/4=256$ 个块号，但 Unix 取 $N=100$。

### 2. 内存开销

- 超级块中的栈空间：$4 \times N + 4 \text{ B（计数器）}$
- $N=100$ 时仅需约 **404 B** 内存

### 3. 极端场景分析

**场景：连续分配大量块**

假设磁盘有 $M$ 块，$N=100$，组数 $G = \lceil M/100 \rceil$：
- 每分配 100 块时，有一次读盘操作（读下一组索引块）
- 平均每块分配需要 $1 + 1/100 \approx 1.01$ 次磁盘 I/O（如果第一次读索引块）
- 在内存栈未耗尽时，**零磁盘 I/O** 即可完成分配

## 典型例题

**例 1**：Unix 成组连接法，$N=100$，超级块中 `s_nfree=5`，栈中块号为 [300, 299, 298, 297, 296]。连续分配 5 个盘块，写出每次分配后栈的状态。

解：
- 分配 1：`s_nfree=4`，栈 = [300, 299, 298, 297]，返回 296
- 分配 2：`s_nfree=3`，栈 = [300, 299, 298]，返回 297
- 分配 3：`s_nfree=2`，栈 = [300, 299]，返回 298
- 分配 4：`s_nfree=1`，栈 = [300]，此时最后一个——注意 300 是索引块
  - 读出 300 号块内容，假设读到下一组空闲块号 [400, 399, 398, …, 301]
  - 新栈：`s_nfree=99`（N-1 块），栈内容 = [400, 399, 398, …, 302]
  - 返回 301
- 分配 5：`s_nfree=98`，栈 = [400, 399, 398, …, 303]，返回 302

**例 2**：在例 1 的基础上，释放块号 296（假设释放前 `s_nfree=98`，栈未满）。写出释放后的栈状态。

解：
- 检查 `s_nfree=98 < 100`（栈未满）
- 压栈：`s_free[98] = 296`
- `s_nfree = 99`
- 栈 = [400, 399, …, 303, 296]

## 易错点

1. **索引块本质是数据块但不是空闲块**：每组的最后一个盘块号存的是下一组的索引，这块盘块已被占用（存索引），不应当做空闲块分配出去
2. **分配时 `s_nfree == 1` 的特殊处理**：此时不能直接弹栈返回，而要读盘再分配
3. **回收时 `s_nfree == N` 的特殊处理**：此时不能直接压栈，要先将当前栈内容写入回收块（新的索引块），然后重新初始化栈
4. **初始化的特殊情况**：系统首次挂载时，需要扫描磁盘构造空闲栈
5. **崩溃恢复**：超级块是内存中的数据结构，若系统崩溃，超级块中的栈内容丢失。Unix 通过 `sync` 系统调用定期将超级块写回磁盘

## 关联知识

- 与「目录项、FCB、簇号与盘块大小的关系」同属**文件系统空间管理**主题
- FAT 文件系统的 FAT 表也可以看作一种空闲管理，区别在于 FAT 管理的是已分配空间的链表，而成组连接法管理的是空闲空间
- 超级块本身也是文件系统的关键元数据区域（通常位于分区第 2 扇区）
