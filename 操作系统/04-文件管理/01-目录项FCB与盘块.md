# 目录项、FCB、簇号与盘块大小的关系

## 定义

### 目录项与 FCB

- **目录项（directory entry）**：目录文件中用于记录一个文件/子目录元信息的基本单元。每个文件或子目录在所属目录中有唯一对应的目录项。
- **FCB（File Control Block，文件控制块）**：OS 管理文件所需的全部元数据——文件名、文件类型、文件物理位置（盘块号/簇号/索引节点号）、存取权限、时间戳、文件大小等。它是文件实体的"身份凭证"。
- **盘块（disk block）**：文件系统读写磁盘的最小逻辑单元。

### 簇（Cluster）

**簇（cluster）** 是文件系统进行**空间分配**的基本逻辑单位，由**一组连续的盘块**组成。

- 在早期的 FAT 文件系统中，文件系统不直接按盘块分配空间，而是按**簇**分配
- 簇大小 = 盘块大小 × 每个簇包含的盘块数（每簇盘块数 = 2^n，如 1、2、4、8…）

**簇号（cluster number）** 就是给每个簇按顺序编的号，FAT 表通过簇号构成一个链表，记录每个文件占用了哪些簇。

> 核心关系：**FCB → 起始簇号 → FAT 表查链 → 得出文件全部盘块位置**

## 核心定理 / 性质

### 1. FCB、盘块、簇的关系图

```
┌─────────────────────────────────────────────┐
│                 目录项（FCB）                   │
│  文件名 | 大小 | 时间 | 起始簇号 | 权限...      │
└───────────────────┬─────────────────────────┘
                    │ 起始簇号 = 3
                    ▼
┌─────────────────────────────────────────────┐
│                 FAT 表                         │
│  簇号   │  下一个簇号                           │
│  0      │  EOF (保留)                         │
│  1      │  EOF (保留)                         │
│  2      │  4     ← 文件首簇                     │
│  3      │  5                                   │
│  4      │  7                                   │
│  5      │  EOF  ← 文件尾                       │
└─────────────────────────────────────────────┘
```

### 2. 关键参数与公式

设：
- 盘块大小 = $B$（通常 512 B）
- 每簇盘块数 = $C$（通常 $C = 2^k$）
- 簇大小 = $B \times C$
- FAT 表项大小 = $E$（12/16/32 bit，对应 FAT12/16/32）

**每簇盘块数**由分区大小决定：

| 分区大小 | 每簇盘块数 | 簇大小 |
|---|---|---|
| ≤ 32 MB | 1 | 512 B |
| 33~64 MB | 2 | 1 KB |
| 65~128 MB | 4 | 2 KB |
| 129~256 MB | 8 | 4 KB |
| 257~512 MB | 16 | 8 KB |
| 513~1024 MB | 32 | 16 KB |
| 1~2 GB | 64 | 32 KB |
| 2~4 GB | 128 | 64 KB |

**FCB 大小**（FAT 系列示例）：

| 属性 | 字节数 |
|---|---|
| 文件名（8.3 格式） | 11 B |
| 属性 | 1 B |
| 保留 | 10 B |
| 时间 | 2 B |
| 日期 | 2 B |
| 起始簇号 | 2 B |
| 文件大小 | 4 B |
| **合计** | **32 B** |

**FAT 表大小**：
```
FAT 表项数 = 分区总簇数
FAT 表大小 = 总簇数 × 每表项字节数
```

**可管理的最大分区**：
```
FAT12：2^12 × 簇大小（通常上限 32 MB）
FAT16：2^16 × 簇大小（通常上限 2 GB）
FAT32：2^28 × 簇大小（通常上限 2 TB）
```

### 3. 簇号与盘块号的换算

$$
\text{起始盘块号} = \text{起始簇号} \times \text{每簇盘块数}
$$

$$
\text{文件占用的盘块数} = \text{文件占用的簇数} \times \text{每簇盘块数}
$$

## 计算方法 / 技巧

### 1. 从 FCB 到文件的完整盘块位置

1. 从 FCB 读出**起始簇号** $cluster_{start}$
2. 从簇号查 **FAT 表**，顺着链表找到所有簇
3. 将每个簇号换算为盘块号：$block = cluster \times blocks\_per\_cluster$
4. 计算实际的物理地址

### 2. 文件大小与簇数的关系

- 文件的逻辑大小 ≠ 占用空间大小
- **簇内碎片**：最后一个簇通常用不满，但整个簇全部分配
- 实际占用空间 = 分配簇数 × 簇大小

$$
\text{文件占用的簇数} = \left\lceil \frac{\text{文件大小}}{\text{簇大小}} \right\rceil
$$

### 3. FAT 损坏的影响范围

- FAT 表中某个簇号损坏 → 该簇以后的整个文件链断裂
- 通常 FAT 表保留两份副本（FAT1、FAT2）做冗余

## 典型例题

**例 1**：某 FAT32 文件系统盘块大小 512 B，每簇 4 个盘块。文件 B 的逻辑大小为 5000 B，问实际占用多少磁盘空间？

解：
簇大小 = 512 × 4 = 2048 B = 2 KB。
文件占用的簇数 = ⌈5000 ÷ 2048⌉ = ⌈2.44⌉ = 3 簇。
实际占用空间 = 3 × 2048 = 6144 B。
簇内碎片 = 6144 − 5000 = 1144 B。

**例 2**：某文件目录项的 FCB 大小为 32 B，起始簇号为 5，每簇 8 个盘块，盘块大小 512 B。FAT 表项如下：FAT[5]=8, FAT[8]=12, FAT[12]=EOF。问文件占用了哪些盘块？

解：
每簇盘块数 = 8，盘块大小 = 512 B。
簇 5 → 盘块 40~47
簇 8 → 盘块 64~71
簇 12 → 盘块 96~103
共 3 簇，24 个盘块。

**例 3**：FAT16 文件系统，盘块大小 512 B，分区大小为 512 MB。问 FAT 表项总数和 FAT 表大小？

解：
512 MB = 512 × 1024 × 1024 B。
已知 FAT16 每簇盘块数约为 16（查表）。
簇大小 = 512 × 16 = 8192 B = 8 KB。
总簇数 = 512 MB ÷ 8 KB = 65536。
FAT16 每表项 2 B。
FAT 表大小 = 65536 × 2 = 131072 B = 128 KB。

## 易错点

1. **盘块 vs 簇**：盘块是底层读写单元，簇是分配单元。一个簇里即使只存 1 字节，也占整个簇的空间（簇内碎片）
2. **磁盘地址换算**：簇号不是直接盘块号，必须乘以每簇盘块数才能得到起始盘块号
3. **FAT 表保留项**：0 号簇和 1 号簇是保留项（分别表示空闲和坏簇标记），文件数据从 2 号簇开始分配
4. **FAT32 最大簇数**：虽然 FAT32 用 32 位编号，但高 4 位保留，实际有效位 28 位
5. **格式化时簇大小的选择**：簇太小 → FAT 表过大；簇太大 → 簇内碎片严重。这是空间与性能的权衡
