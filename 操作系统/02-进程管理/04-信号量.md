# 信号量

## 定义

信号量（Semaphore）是 Dijkstra 在 1965 年提出的同步原语，用一个整型变量 $S$ 和两个原子操作（P 和 V）来实现进程间的互斥与同步。

```c
P(S) {  // wait / proberen（荷兰语"测试"）
    while (S <= 0);   // 忙等（不阻塞）或 阻塞等待（取决于实现）
    S = S - 1;
}

V(S) {  // signal / verhogen（荷兰语"增加"）
    S = S + 1;
}
```

> 信号量中的"信号"来自铁路信号灯——一根柱子升起（V）表示"可以通过"，放下（P）表示"等待"。记忆技巧：P = 要过检查点（需要资源），V = 释放信号（归还资源）。

---

## 三种信号量

| 类型 | S 的取值范围 | 用途 |
|---|---|---|
| 整型信号量 | $S \geq 0$（不阻塞，忙等） | 基本互斥，浪费 CPU |
| 记录型信号量 | $S$ 可为负（$|S|$ = 等待队列长度） | 真正的阻塞/唤醒，实用 |
| 二值信号量（互斥锁） | $S \in \{0, 1\}$ | 纯互斥 |

### 记录型信号量（408 默认）

```c
typedef struct {
    int value;
    struct process *L;  // 等待队列
} semaphore;

void P(semaphore *S) {
    S->value--;
    if (S->value < 0) {
        block(S->L);   // 阻塞当前进程，放入等待队列
    }
}

void V(semaphore *S) {
    S->value++;
    if (S->value <= 0) {
        wakeup(S->L);  // 从等待队列唤醒一个进程
    }
}
```

**关键理解**：
- `S->value > 0`：表示还有多少个资源可用
- `S->value = 0`：资源刚好用完，无等待进程
- `S->value < 0`：`|S->value|` 就是当前阻塞等待的进程数量

## 信号量的两种用途

### 互斥（同一进程内 P 和 V 成对）

```c
semaphore mutex = 1;  // 初值为 1，表示"允许一个进程进入"

P(mutex);
    临界区
V(mutex);
```

### 同步（P 和 V 在不同进程中成对）

```c
semaphore s = 0;  // 初值为 0，表示"B 等 A 后执行"

进程 A:           进程 B:
  操作前半段         P(s);
  V(s);             操作后半段
```

> 互斥的 P/V 在**同一个进程**内配对；同步的 P/V 在**不同进程**间配对。这是区分互斥和同步的标志。

## 典型例题

**例**：三个进程 A、B、C 共享一个大小为 1 的缓冲区。A 往缓冲区放数据，B 和 C 都从缓冲区取数据（一个数据只取一次），且要求 A 放入后 B 或 C 才能取。用信号量实现。

解：需要两个信号量：

```c
semaphore empty = 1;   // 缓冲区空，A 可以放
semaphore full  = 0;   // 缓冲区满，B 或 C 可以取
semaphore mutex = 1;   // B 和 C 互斥地访问缓冲区

A:  P(empty);  放入;  V(full);
B:  P(full);   P(mutex);  取出;  V(mutex);  V(empty);
C:  P(full);   P(mutex);  取出;  V(mutex);  V(empty);
```

关键：`full` 初值为 0，迫使 B/C 必须等 A 放入后才能取。取出后 V(empty) 告知 A"可以放新数据了"。`mutex` 保证 B 和 C 不会同时取到同一个数据。

## 易错点

1. **P 操作中的 `S--` 和判断不是原子的一步**：必须用硬件原语（如 test-and-set 或关中断）保证整个 P/V 操作不可分割。考试中不必关心具体硬件实现，当作原子即可。

2. **死锁风险**：`P(s1); P(s2); ... V(s2); V(s1);` 和另一个进程的 `P(s2); P(s1); ...` 会死锁。多个信号量时的 P 顺序必须全局一致。

3. **`S <= 0` 时唤醒不是 `S < 0`**：V 唤醒的条件是 `S <= 0`（不是 `S < 0`）。因为 P 中先 `S--` 再判断，所以 $S=0$ 意味着"减之前是 1，资源分配给了当前进程，没有等待者"——这是临界情况，不需要唤醒。

4. **整型信号量 P 的 `while(S<=0)` 是忙等**：CPU 空转浪费资源。记录型信号量用 `block()` 让进程进入阻塞态，让出 CPU，解决了忙等问题。
