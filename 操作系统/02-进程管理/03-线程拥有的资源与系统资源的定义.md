# 线程拥有的资源与系统资源的定义

## 线程"几乎不占资源"——那它到底占什么

每个线程**独立拥有**：

| 资源 | 说明 |
|---|---|
| 线程 ID | 操作系统内唯一标识 |
| 程序计数器（PC） | 记录自己执行到哪条指令 |
| 寄存器上下文 | 通用寄存器、状态寄存器的值，切换时保存/恢复 |
| 栈 | 每个线程独享一个栈（存放局部变量、函数调用帧） |
| 线程状态 | 就绪/运行/阻塞等 |
| 线程控制块（TCB） | OS 管理该线程的数据结构 |

与同进程的其他线程**共享**：代码段、数据段（全局变量）、堆、打开的文件、信号处理器、当前工作目录、用户 ID。

## "系统资源"到底指什么

OS 层面以**进程**为单位分配的资源包括：

| 资源类别 | 具体内容 | 线程之间 |
|---|---|---|
| 地址空间 | 代码段、数据段、堆、共享库映射区 | 共享（同一虚拟地址空间） |
| 打开文件 | 文件描述符表、文件偏移量指针 | 共享（操作同一张表） |
| I/O 设备 | 打印机、网卡等独占设备 | 共享 |
| 内存配额 | 进程的整个虚拟地址空间（页表） | 共享 |
| PCB（进程控制块） | 进程 ID、优先级、资源清单、统计信息 | 只有一份 |
| 信号 | 挂起的信号及处理设置 | 共享 |
| 用户凭证 | UID、GID | 共享 |

> "线程不占系统资源"的意思是线程不额外占据**以上这些东西**——它们不独立拥有地址空间、不独立持有文件表、不单独分配内存配额。但每个线程的 TCB、栈空间和寄存器上下文仍然占用实际内存（通常一个线程的栈为几 MB），所以"几乎不占"不是"不占"，而是"相比进程少得可以忽略"。

## 典型例题

**例**：同一进程的多个线程，以下哪些是各自独立的？

A. 栈　　B. 堆　　C. PC　　D. 文件描述符表

答：A 和 C。栈和 PC（以及寄存器上下文）是线程私有的；堆和文件描述符表是同一进程内所有线程共享的。
