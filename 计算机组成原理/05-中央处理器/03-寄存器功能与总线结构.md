# CPU 核心寄存器功能与总线结构

## 定义

CPU 内部寄存器是位于控制器和运算器中的高速存储单元，各自承担指令执行流程中的特定角色。以下为五个核心寄存器：

| 寄存器 | 全称 | 所属部件 | 核心功能 |
|---|---|---|---|
| PC | Program Counter 程序计数器 | 控制器 | 存放下一条待取指令的地址，具有计数功能（PC+1） |
| MAR | Memory Address Register 存储器地址寄存器 | 控制器 | 存放当前访存的地址（读或写），其输出直接连地址总线 |
| MDR | Memory Data Register 存储器数据寄存器 | 控制器 | 存放从内存读出或即将写入内存的数据，其输入/输出连数据总线 |
| IR | Instruction Register 指令寄存器 | 控制器 | 存放当前正在执行的指令码，其操作码部分送译码器 |
| ACC | Accumulator 累加器 | 运算器 | 存放 ALU 的一个源操作数及运算结果，是运算器的核心数据寄存器 |

---

## 核心定理 / 性质

### 单总线数据通路结构

下面是经典单总线 CPU 结构中五个寄存器在数据通路内的位置与连接关系。理解这张图就理解了指令执行的全部硬件细节。

```
                    ┌─────────── CPU 内部总线 ───────────┐
                    │          (单总线，分时复用)           │
                    │                                      │
    ┌──────┐   ┌────┴────┐   ┌──────┐   ┌──────┐   ┌────┴─────┐
    │  PC  │   │   MAR   │   │ MDR  │   │  IR  │   │   ACC    │
    │      │   │         │   │      │   │      │   │(累加器)   │
    └──┬───┘   └────┬────┘   └──┬───┘   └──────┘   └────┬─────┘
       │            │           │                       │
       │   +1       │           │                       │
       └────────────┘           │              ┌────────┴──────┐
                                │              │      Y        │(暂存器)
    ┌───────────────────────────┼──────────────┤               │
    │       地址总线             │   数据总线    │   ┌─────┐     │
    │    ─────────────────────► │ ◄──────────► │   │ ALU │     │
    │                           │              │   └──┬──┘     │
    ▼                           ▼              │      │        │
 ┌──────┐                  ┌──────────┐        │   ┌──┴──┐     │
 │ 主存 │                  │  Cache   │        │   │  Z  │     │(输出暂存)
 │(RAM) │                  │(高速缓存)│        │   └──┬──┘     │
 └──────┘                  └──────────┘        │      │        │
    ▲                           ▲              └──────┴────────┘
    │                           │
    └───── 地址/数据总线 ────────┘
```

### 寄存器的输入/输出控制信号

每个寄存器的数据流入流出由控制器产生的微操作信号精确控制：

| 寄存器 | 数据来源（输入） | 数据去向（输出） | 关键控制信号 |
|---|---|---|---|
| PC | 内部总线（跳转时从 IR 或 ALU），或 PC+1 | 内部总线→MAR（取指时） | PC→BUS, BUS→PC, PC+1 |
| MAR | 内部总线（来自 PC 或 IR 地址码） | 地址总线→主存/Cache | MAR→ABUS |
| MDR | 数据总线（来自主存/Cache）或内部总线（写操作时） | 内部总线→IR（取指）/→寄存器（取数），数据总线→主存 | MDR→BUS, BUS→MDR, MDR→DBUS |
| IR | 内部总线（来自 MDR，取指时） | 操作码→译码器，地址码→内部总线 | BUS→IR |
| ACC | ALU 输出（经过 Z）或内部总线 | 内部总线→MDR（存数），Y→ALU | BUS→ACC, ACC→BUS, Z→ACC |

## 计算方法 / 技巧

### 与 Cache 的连接方式

现代 CPU 中 Cache 位于 CPU 和主存之间的数据通路上，根据 Cache 组织形式不同，连接方式分为：

**1. 分离 Cache 结构（Harvard 架构在 CPU 核内）**：
```
        PC ──► I-Cache(指令Cache) ──► 主存
        MAR ──► D-Cache(数据Cache) ──► 主存
```
- I-Cache 从 PC 提供的地址取指令，经数据通路送入 MDR→IR
- D-Cache 从 MAR 提供的地址读/写数据，经数据通路与 MDR 交互
- 优点：取指和取数可并行，消除结构冒险

**2. 统一 Cache 结构**：
```
        PC ──► MAR ──► 统一Cache ──► 主存
                         │
                         └──► MDR ──► CPU内部总线
```
- 指令和数据共享同一个 Cache
- 优点：硬件简单，动态分配容量
- 缺点：取指与取数可能冲突（结构冒险）

**3. 多级 Cache 层次**：
```
CPU ←→ L1(I/D分离) ←→ L2(统一) ←→ L3(统一) ←→ 主存
```
- L1 Cache 通常内嵌于 CPU 核，延迟 1-2 个时钟周期
- MAR 发出的地址依次查询 L1→L2→L3，命中即止
- PC 取指时地址通过 MAR 送到 I-Cache；LOAD/STORE 时送到 D-Cache

### 寄存器间的数据传输时间

在单总线结构下，每次内部总线传输占用一个时钟周期。关键路径分析：

- **取指阶段**：最少 3 个时钟（PC→MAR，Read MEM→MDR→IR，PC+1）
- **间址阶段**：最少 2 个时钟（IR(addr)→MAR，Read MEM→MDR→Y）
- **执行阶段**：视指令类型 1-3 个时钟
- 各阶段间由 CPU 的**时序信号**（节拍发生器）严格节拍控制

## 典型例题

### 例题一：内存取数指令（覆盖取指周期 + 间址周期 + 执行周期）

**指令**：`LOAD ACC, (addr)`  —— 将主存 addr 单元内容取至累加器 ACC

假设指令为单字长，采用**寄存器间接寻址**（地址码给出的是存放有效地址的寄存器号，但本例用直接地址说明），且需经历间址周期（地址码指向的单元存的是有效地址）。

**取指周期（FETCH）**——所有指令共用：

| 时钟 | 微操作 | 数据流 | 说明 |
|---|---|---|---|
| T1 | PC→MAR | PC 的值送入 MAR | MAR 获得待取指令地址 |
| T2 | 1→R, MAR→ABUS | MAR 驱动地址总线；发存储器读命令 | 主存接收地址和读信号 |
| | MEM→DBUS→MDR | 选中存储单元的内容送上数据总线，锁入 MDR | 指令码进入 MDR |
| T3 | MDR→BUS→IR | MDR 数据经内部总线打入 IR | 指令码进入指令寄存器 |
| T4 | PC+1→PC | 程序计数器自增 | PC 指向下一条指令 |

> 取指周期中 PC 同时是"送 MAR"和"自增"。T2 的访存等待时间取决于 Cache 命中与否——若 I-Cache 命中则 1 个时钟；Cache 缺失则需额外多个时钟等待主存（插入等待周期 Tw）。

**间址周期（INDIRECT）**——只有含间接寻址的指令才进入此周期：

| 时钟 | 微操作 | 数据流 | 说明 |
|---|---|---|---|
| T5 | IR(地址码)→BUS→MAR | IR 的地址字段经内部总线打入 MAR | MAR 获得操作数地址的地址 |
| T6 | 1→R, MAR→ABUS | MAR 驱动地址总线 | 发存储器读命令 |
| | MEM→DBUS→MDR | 有效地址读到 MDR | MDR 中现在是真正的操作数地址 |
| T7 | MDR→BUS→MAR | 有效地址送入 MAR | MAR 获得操作数所在地址 |

**执行周期（EXECUTE）**：

| 时钟 | 微操作 | 数据流 | 说明 |
|---|---|---|---|
| T8 | 1→R, MAR→ABUS | MAR 再次驱动地址总线 | 发读命令，地址为有效地址 |
| | MEM→DBUS→MDR | 操作数从主存/Cache 读入 MDR | 此时 D-Cache 查找到 |
| T9 | MDR→BUS→Y | 操作数经内部总线打入暂存器 Y | 为 ALU 运算做准备 |
| | ALU: Y+0→Z | ALU 做直通运算（或不做运算） | 旁路 ALU 将数据传到 Z |
| T10 | Z→BUS→ACC | Z 经内部总线打入 ACC | 操作数最终到达累加器 |

> **Cache 参与的关键时刻**：T2（取指令）、T6（读有效地址）、T8（读操作数）各访问一次存储器。若有分离 I/D-Cache，T2 查 I-Cache，T6/T8 查 D-Cache，各仅 1 个时钟即可完成。

### 例题二：条件转移指令 + 中断（覆盖取指周期 + 执行周期 + 中断周期）

**指令**：`JZ addr` —— 若 ACC = 0，则跳转到 addr（条件成立的情况）

假设在执行周期末尾，外部设备发出了中断请求且 CPU 响应。

**取指周期（FETCH）**：与例题一完全相同（PC→MAR→Read→MDR→IR，PC+1），占用 T1-T4。

**执行周期（EXECUTE）**：

| 时钟 | 微操作 | 说明 |
|---|---|---|
| T5 | ACC→BUS→ALU(测试) | 累加器内容送 ALU 做全零检测 |
| | 若 ACC≠0：执行周期结束，无额外操作 | 条件不成立，顺序执行 |
| | 若 ACC=0：IR(addr)→BUS→PC | 条件成立：地址码直接打入 PC |

> 条件转移指令**无间址周期**。执行周期实际上只需 1-2 个时钟即可完成——ALU 检测 ACC 的全零标志，控制器据此选通 PC 的输入来源。

**中断周期（INTERRUPT）**——当前指令执行完毕后，若 INTR 有效且 IF=1，进入：

| 时钟 | 微操作 | 数据流 | 说明 |
|---|---|---|---|
| T6 | SP-1→SP→MAR | 栈指针减 1 后送 MAR | 准备保存断点（PC当前值） |
| T7 | PC→BUS→MDR | 断点值（已指向下一条指令）送 MDR | MDR 准备好写入数据 |
| | 1→W, MAR→ABUS | 发存储器写命令 | 将断点压入堆栈 |
| | MDR→DBUS→MEM | 写入堆栈 | |
| T8 | SP-1→SP→MAR | 栈指针再减 1 | 准备保存 PSW（程序状态字） |
| T9 | PSW→BUS→MDR | 状态字送 MDR | |
| | 1→W, MAR→ABUS | 发存储器写命令 | |
| | MDR→DBUS→MEM | 写入堆栈 | |
| T10 | 向量地址→BUS→MAR | 中断向量地址送 MAR | 从中断向量表取 ISR 入口 |
| T11 | 1→R, MAR→ABUS | 发存储器读命令 | |
| | MEM→DBUS→MDR | 读得中断服务程序入口地址 | |
| T12 | MDR→BUS→PC | 入口地址写入 PC | 下一条将从 ISR 开始执行 |

> **中断隐指令**（T6-T12）是在硬件层面自动完成的，不通过软件。共 7 个时钟周期。中断周期中 MAR 产生的地址是**栈地址**（T6/T8）和**中断向量表地址**（T10），触发的是 D-Cache 查找（因为是数据访问）。

## 两个例子的机器周期覆盖总览

| 机器周期 | 例题一 LOAD (addr) | 例题二 JZ + 中断 |
|---|---|---|
| 取指周期 | ✔ 经历 | ✔ 经历 |
| 间址周期 | ✔ 经历（读有效地址） | ✘ 无条件转移无限间址 |
| 执行周期 | ✔ 经历（读操作数→ACC） | ✔ 经历（条件判断→PC） |
| 中断周期 | ✘ 无 | ✔ 经历（保存断点→取向量） |

四个机器周期全部覆盖。

## 易错点

1. **MAR 与 MDR 的角色不能互换**：MAR 只与地址总线交互，MDR 只与数据总线交互。MAR 的内容解释为"地址"，MDR 的内容解释为"数据"。取指时 MDR 中的数据本质是指令，但在硬件层面它就是从数据总线进来的比特流，送往 IR 后才被"解释"为指令。

2. **PC+1 不是简单的加法**：在 16/32/64 位字长机器中，"1"的含义是"下一条指令的地址偏移"，在定长指令系统中就是 +4（32位）等。通常由独立的 PC 递增器（incrementer）硬件实现，而非走 ALU。

3. **MDR→IR 与 MDR→ACC 的区别**：同是从 MDR 出来的数据，送 IR（T3）发生在取指周期，送 ACC（T9-T10）发生在执行周期。这是由**时序信号**（节拍）和控制信号共同决定的，硬件用多路选择器（MUX）在内部总线上切换。

4. **单总线结构的瓶颈**：所有寄存器之间的数据流动都争用同一条内部总线，每次只能传一个数据。这就是为什么 `ADD R1, (addr)` 需要多个时钟——每次总线传输都必须安排在不同的节拍。

5. **Cache 对 MAR/MDR 的影响**：MAR/MDR 看到的仍是统一的地址/数据接口——Cache 是**透明的**。区别在于：Cache 命中时 MEM 阶段缩短为 1 个时钟；Cache 缺失时 CPU 通过 WAIT 信号插入等待周期。

6. **中断周期在指令的最后**：中断周期不是在指令执行到一半时插入的，而是在当前指令**完全执行完毕后**、取下一条指令**之前**插入的。CPU 在每条指令末尾检查 INTR 信号。

7. **ACC 不是通用寄存器**：在单总线结构中，ALU 的一个输入固定来自 ACC（通过 Y 暂存），另一个输入来自内部总线。这意味着二元运算必须有一个操作数先放入 ACC。这是单累加器结构的限制（现代 CPU 已多采用通用寄存器组 GPR）。

8. **Y 和 Z 是透明寄存器**：它们不在指令中可寻址，只在微操作层面起作用。Y 的作用是暂存 ALU 的一个输入（因为 ACC 不能同时驱动总线和供给 ALU），Z 暂存 ALU 输出（避免组合逻辑环）。

## MUX（多路选择器）在 CPU 数据通路中的角色

### 定义

多路选择器（Multiplexer，MUX）是组合逻辑电路，功能是**从多个输入中选择一路输出**。$n$ 位选择信号从 $2^n$ 路输入中选通一路。

在单总线 CPU 结构中，MUX 是让一条总线被多个寄存器**分时共享**的关键硬件——它决定"此刻总线上是谁的数据"。

### CPU 数据通路中的四处关键 MUX

**(1) 内部总线入口 MUX（最核心）**

```
 PC ──────────────┐
 ACC ─────────────┤
 MDR ─────────────┼──► MUX ──► 内部总线
 IR(地址码) ──────┤
 ALU(Z) ──────────┘
       ▲
  选择信号（来自控制器）
```

每个寄存器的输出都经过三态门连接到内部总线。控制器在任何时刻只选通**一个**寄存器的输出门，这就是 MUX 的功能——只不过是分布式的。没有这个 MUX，所有寄存器同时驱动总线，数据冲突。

**(2) MAR 入口 MUX**

```
PC ──────────────┐
IR(地址码) ──────┼──► MUX ──► MAR
SP ──────────────┘
```

不同周期里 MAR 的输入来源不同：取指周期来自 PC，间址周期来自 IR 的地址码，中断周期来自 SP。MUX 在控制器指挥下切换通路。

**(3) PC 入口 MUX**

```
PC+1 ────────────┐
IR(跳转地址) ────┼──► MUX ──► PC
MDR(中断向量) ───┘
```

下一条指令地址有三种来源：顺序执行（PC+1）、跳转（IR 中的地址码）、中断返回（从中断向量表读入的地址）。MUX 三选一。

**(4) ALU 输入端 MUX**

ALU 的一个输入固定来自 ACC（经过 Y 暂存），另一个输入来自内部总线（可能是 MDR 的数值或 IR 的立即数）。MUX 选择"此刻和 ACC 一起做运算的是什么"。

### 关键特性

MUX 是组合逻辑，不是时序器件——选择信号一变，输出立刻跟着变。这意味着：**同一时钟周期内，MUX 的选择信号必须先稳定，被选中的数据才能正确传过去**。这也正是微操作必须分拍的根源之一——前一拍让选择信号稳定，后一拍才让被选中通路上的数据流动。
