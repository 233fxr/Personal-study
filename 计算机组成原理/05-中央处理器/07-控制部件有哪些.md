# 控制部件有哪些

## 定义

CPU 由**数据通路**（运算器）和**控制器**两大部分组成。控制部件的职责是：根据当前指令的操作码和状态标志，在正确的时刻发出正确的控制信号，指挥数据通路完成指令执行。

---

## 控制部件的清单

### 所有控制器共有的部件

| 部件 | 功能 |
|---|---|
| PC（程序计数器） | 存放下一条指令地址，具有计数功能 |
| IR（指令寄存器） | 存放当前正在执行的指令码 |
| 指令译码器（Decoder） | 对 IR 中的操作码译码，识别指令类型 |
| 时序发生器（Timing Generator） | 产生节拍信号（T0, T1, T2...）和机器周期标志（FE, IND, EX, INT），决定"现在处于哪个阶段" |
| 微操作信号发生器 | 输入端：操作码（来自译码器）+ 节拍（来自时序发生器）+ 状态标志（来自 PSW）；输出端：所有控制信号（如 PCout, MARin, ALU_ADD, MEM_Read 等） |
| 中断逻辑 | 检测中断请求信号 INTR，在当前指令末尾决定是否进入中断周期 |

### 微程序控制器额外需要

| 部件 | 功能 |
|---|---|
| 控制存储器 CM | CPU 内部 ROM，存储所有微程序 |
| μPC（微程序计数器） | 指向 CM 中下一条微指令的地址 |
| CMDR（控制存储器数据寄存器，即 μIR） | 存放当前取出的微指令 |

### 硬布线控制器不需要 CM

硬布线控制器不存在控制存储器和 μPC——微操作信号发生器**直接用组合逻辑电路**根据操作码和节拍产生控制信号，不需要从 CM 中取微指令。

---

## 核心定理 / 性质

### 控制信号的来源模型

```
        ┌──────────┐     ┌───────────┐
        │  IR(操作码) │────►│  指令译码器  │
        └──────────┘     └─────┬─────┘
                               │  "这是 ADD 指令"
                               ▼
        ┌──────────┐     ┌───────────┐
        │  时序发生器  │────►│ 微操作信号  │────► 所有控制信号
        │  T1, T2... │     │  发生器    │     (PCout, MARin,
        └──────────┘     └─────┬─────┘      ALU_ADD, MEM_Read...)
                               ▲
        ┌──────────┐          │
        │ PSW(标志)  │─────────┘
        │ ZF, CF... │  "结果是零，影响下一条条件转移"
        └──────────┘
```

三种输入共同决定控制信号的输出：
- **操作码**（来自 IR）：决定「要做什么」
- **节拍**（来自时序发生器）：决定「现在做哪一步」
- **状态标志**（来自 PSW）：决定「出现特殊情况怎么处理」

---

## 易错点

1. **IR 既是数据通路又是控制部件**：IR 锁存指令码（数据通路角色），同时它驱动译码器（控制部件角色）。所以两种归类都对。

2. **PC 属于控制器而非运算器**：PC 决定指令执行的先后顺序，不参与数据运算，归属于控制器。这一定位在硬布线和微程序两种实现方式下都一样。

3. **时序发生器不是时钟本身**：CPU 外部晶振提供时钟脉冲，时序发生器将其分频并产生带标记的节拍序列（T0→T1→T2→T3→T0...），并组合出机器周期信号（取指周期 / 间址周期 / 执行周期 / 中断周期）。
