# CISC 与 RISC

## 定义

- **CISC**（Complex Instruction Set Computer，复杂指令集计算机）：指令系统庞大，单条指令功能强，一条指令可完成复杂的多步操作。代表：x86。
- **RISC**（Reduced Instruction Set Computer，精简指令集计算机）：指令系统精简，每条指令功能单一，靠多条简单指令的组合完成复杂任务。代表：ARM、MIPS、RISC-V。

---

## 核心对比

| | CISC | RISC |
|---|---|---|
| 指令数量 | 多（数百条） | 少（通常几十条） |
| 指令长度 | 变长（1~15 字节） | 定长（通常 4 字节） |
| 指令格式 | 多样、不规则 | 少而规整 |
| 寻址方式 | 多（十几种） | 少（3~5 种） |
| 访存指令 | 运算指令也可访存 | 仅 LOAD/STORE 可访存 |
| 寄存器数量 | 少（x86 通用寄存器仅 8 个） | 多（通常 32 个通用寄存器） |
| 控制器 | 多为微程序控制 | 多为硬布线控制 |
| 流水线实现 | 困难（变长、复杂） | 容易（定长、规整） |
| 编译器设计 | 较简单（指令功能强） | 难度较大（需优化指令调度） |
| 典型代表 | x86（Intel/AMD） | ARM、MIPS、RISC-V |

---

## 核心定理 / 性质

### RISC 得以成立的依据——"二八定律"

实际程序中，80% 的时间在执行 20% 的简单指令（如 MOV、ADD、CMP、JMP）。CISC 那些复杂指令虽然功能强大，但使用频率极低，却使得整个控制器的设计变得极其复杂、流水线难以优化。RISC 的思路是：砍掉那 80% 低频复杂指令，将省下的晶体管用来做更多通用寄存器和更高效的流水线。

### 关键区别：LOAD/STORE 架构

RISC 的核心特征之一是**运算指令只能操作寄存器，不能直接访存**：
- `ADD R1, R2, R3` —— RISC 做法，三个操作数全在寄存器中
- `ADD R1, (addr)` —— CISC 允许的操作，操作数可以直接来自内存

RISC 要完成同样的功能，必须先用 LOAD 把内存数据取到寄存器，运算后再用 STORE 写回。单条指令的指令数多了，但每条指令执行快且可流水，整体吞吐量更高。

### 现代 CPU 的实际做法：融合

| CPU | 外部（指令集） | 内部（微架构） |
|---|---|---|
| Intel Core / AMD Ryzen | CISC（x86 指令集） | RISC 微操作核心（解码器将 x86 指令翻译为 μOps） |
| Apple M 系列 | RISC（ARM 指令集） | RISC 微架构 |

> 纯 CISC 和纯 RISC 的边界早已模糊。现代 x86 CPU 在硬件解码阶段将复杂指令拆成多条类似 RISC 的微操作（μOps），然后用 RISC 风格的超标量流水线去执行它们。外表是 CISC，内脏是 RISC。

## 典型例题

**例**：下列特征中，哪些属于 RISC？

① 指令长度固定  ② 寻址方式丰富  ③ 只有 LOAD/STORE 指令访存  ④ 通用寄存器多  ⑤ 采用微程序控制

答：①③④ 属于 RISC。② 是 CISC 特征；⑤ 是 CISC 特征（RISC 多为硬布线控制）。

## 易错点

1. "精简"不是指令数量少到不够用——RISC 的几十条指令经过精心挑选，覆盖了 99% 的程序需求。省掉的是那些"牛刀杀鸡"的复杂指令（如 x86 的字符串操作指令 `REP MOVSB`）。

2. ARM 不是一定比 x86 省电——功耗差异主要来自微架构设计目标和制程工艺，而非指令集本身。Apple M 系列高性能核心的功耗并不比同级 x86 低很多。

3. x86 变长指令的"变长"是 CISC 流水线实现困难的根本原因——取指阶段不知道当前指令到底多长，必须等到部分译码后才能确定下一条指令的起始位置，无法简单并行的取多条指令。

4. RISC-V 是开源指令集，不等于 RISC-V CPU 本身开源——指令集规范（ISA）开源免费，但具体的 CPU 微架构设计是各家厂商的商业秘密。
