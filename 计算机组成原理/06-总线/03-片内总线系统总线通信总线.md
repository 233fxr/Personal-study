# 片内总线、系统总线、通信总线

## 定义

按总线所处的**物理位置和覆盖范围**，分为三个层次：

```
┌─────────────────────────────────────────────────┐
│                 CPU 芯片                         │
│  ┌───┐  ┌───┐  ┌───┐                            │
│  │ PC │  │ACC│  │ALU│  ←── 片内总线（On-Chip）    │
│  └──┬─┘  └──┬┘  └──┬┘      最短（微米~毫米）       │
│     └───────┼──────┘       最快（GHz级）           │
│             │                                     │
└─────────────┼─────────────────────────────────────┘
              │
    ┌─────────┼─────────────────────────┐
    │         系统总线（System Bus）       │
    │         CPU ←→ 主存 ←→ I/O         │
    │         距离：厘米级                  │
    │         速度：百MHz~GHz              │
    └─────────┼─────────────────────────┘
              │
    ┌─────────┼─────────────────────────┐
    │        通信总线（Communication Bus） │
    │        计算机 ←→ 外设 / 其他计算机    │
    │        距离：米~千米                 │
    │        速度：Mbps~Gbps              │
    └───────────────────────────────────┘
```

---

## 核心对比

| | 片内总线 | 系统总线 | 通信总线 |
|---|---|---|---|
| 位置 | CPU 芯片内部 | 主板（PCB） | 机箱外 / 机箱内设备间 |
| 连接对象 | 寄存器、ALU、Cache、控制器 | CPU ↔ 主存 ↔ I/O 接口 | 计算机 ↔ 外设 / 网络 / 其他计算机 |
| 物理距离 | 微米 ~ 毫米 | 几厘米 ~ 十几厘米 | 几十厘米 ~ 几千米 |
| 速度 | 最高（CPU 主频级，GHz） | 中等（百 MHz ~ GHz） | 较低（Mbps ~ 数十 Gbps） |
| 宽度 | 很宽（32~256 位甚至更宽） | 较宽（32~64 位） | 较窄（1 位串行或几通道差分对） |
| 定时方式 | 同步（CPU 时钟统一控制） | 同步或异步 | 异步为主 |
| 典型例子 | CPU 内部数据通路 | 地址/数据/控制三总线 | USB、PCIe、SATA、Ethernet |
| 标准 | 无公开标准（各厂商自定） | 有标准（如 Intel Front-Side Bus） | 国际标准（USB-IF、PCI-SIG 等） |

## 核心性质

### 片内总线

就是前面笔记中反复出现的 **CPU 内部单总线结构**（或多总线结构）。所有寄存器都挂在上面，每个时钟周期由控制器选通一路数据传输。设计目标是**极致带宽**——内部总线宽度不受芯片引脚数限制（芯片内部走线成本远低于引出引脚），可以做得非常宽。

### 系统总线

即经典的地址总线 + 数据总线 + 控制总线，是 CPU 对外通信的唯一通道。随着技术进步，传统并行的三总线结构在多核时代逐渐被**串行点对点**取代（如 Intel QPI、AMD Infinity Fabric），但 408 考研以并行的三总线为教学模型。

### 通信总线

连接计算机与外部世界的总线。按传输方式分为：

| | 串行通信总线 | 并行通信总线 |
|---|---|---|
| 数据线 | 1 根（或差分对） | 多根同时传 |
| 速度 | 理论上慢，实际上因无偏移问题可以跑很高频率 | 理论上快，高频下信号偏移限制了速度 |
| 距离 | 远 | 近（并行线间串扰严重） |
| 目前趋势 | **绝对主流**（USB、SATA、PCIe、Ethernet） | 基本淘汰（老式并口、IDE 硬盘线） |

## 易错点

1. **PCIe 是通信总线不是系统总线**：PCIe 连接 CPU 和显卡/SSD，物理上插在主板上，看起来像系统总线——但它走的是串行差分信号，有热插拔能力，标准化程度和系统总线完全不同。408 考试的"系统总线"专指那套地址/数据/控制并行三总线模型。

2. **片内总线和系统总线不是"快慢"一个维度能比较的**：片内总线宽度可以 256 位×5GHz = 1.28Tbps，系统总线 64 位×100MHz = 6.4Gbps，差了两个数量级。这正是 Cache 存在的根本原因——内外速度鸿沟太大。

3. **不要混淆"内部总线"和"片内总线"**：有些教材把 CPU 内部的称为"片内总线"，把 CPU 到主存的称为"系统总线"。统一说法就是：片内 = CPU 芯片内部，系统 = 主板上，通信 = 出机箱。

4. **三种总线没有物理上的"中继"关系**：不是片内总线出来接到系统总线上、系统总线再接到通信总线上。CPU 通过系统总线的 I/O 接口连接到通信总线控制器（如 USB 控制器），通信总线控制器再驱动外部总线。中间有桥接芯片，不是直连。
