# 总线连接方式与传输过程

## 定义

计算机的部件不是胡乱连线的——每一段物理通路都由特定类型的总线承载：

| 总线类型 | 连接范围 | 传输内容 | 方向 |
|---|---|---|---|
| CPU 内部总线 | CPU 内部：PC、MAR、MDR、IR、ACC、ALU 之间 | 数据（指令码、操作数、地址） | 双向，分时复用 |
| 地址总线 | CPU(MAR) → 主存 / I/O 端口 | 内存地址或 I/O 端口号 | 单向（CPU 发出） |
| 数据总线 | CPU(MDR) ↔ 主存 ↔ I/O 接口 | 指令、数据、状态信息 | 双向 |
| 控制总线 | CPU ↔ 各部件 | 读/写信号、中断请求、总线请求/响应、时钟等 | 每根线方向固定 |

> 系统总线 = 地址总线 + 数据总线 + 控制总线，是 CPU 对外通信的唯一通道。

---

## 不同部件的总线连接示例

### 示例一：CPU 与主存的连接

```
                 ┌─── 地址总线（单向，CPU→主存）───┐
                 │                                  │
    ┌──────┐    │    ┌──────────────────────┐       │
    │ CPU  │ MAR├────┤ 地址译码器 → 存储矩阵  │       │
    │      │    │    │                      │       │
    │      │ MDR├────┤ 数据缓冲 ←→ 存储矩阵  │       │
    │      │    │    └──────────────────────┘       │
    │      │    │          主存                      │
    │      │    ├─── 数据总线（双向）───────────────┘
    │      │    ├─── 控制总线 ──────────────────────►
    │      │    │   (MEM_Read, MEM_Write)
    └──────┘    │
```

- 地址总线宽度 = MAR 的位数，决定最大寻址空间
- 数据总线宽度 = MDR 的位数，决定一个存储周期读写多少位
- 控制总线中的 MEM_Read 和 MEM_Write 由 CPU 控制器在对应机器周期发出

### 示例二：CPU 与 I/O 设备的连接

```
    CPU ←──系统总线──→ I/O 接口 ←──设备专用线──→ 外设
                         │
                    含有：数据缓冲寄存器
                          状态寄存器（设备就绪/忙）
                          控制寄存器（启动/停止命令）
```

I/O 接口插在 CPU 和外部设备之间，解决速度匹配、信号格式转换、中断管理等问题。CPU 不直接跟外设打交道——它通过系统总线读写 I/O 接口中的**端口寄存器**来控制外设。

### 示例三：DMA 方式下的连接

```
    CPU ──┬── 系统总线 ──┬── 主存
          │              │
          │         ┌────┴────┐
          │         │   DMA    │
          └─────────┤ 控制器   ├── I/O 设备
                    └─────────┘
```

DMA 控制器也是系统总线上的主设备。平时 CPU 使用总线；DMA 传送时，DMA 控制器向 CPU 发出**总线请求（HOLD / BR）**，CPU 响应后交出总线控制权（发出 HLDA / BG），DMA 控制器接管地址总线和控制总线，直接驱动主存的读写操作。传送结束后 DMA 撤销请求，CPU 收回总线。

## 不同类型传输的总线工作过程

### 类型一：CPU 读主存（取指令 / LOAD）

以取指周期为例，系统总线的操作序列：

| 步骤 | 总线动作 | 说明 |
|---|---|---|
| ① | CPU 内部：PC→内部总线→MAR | 指令地址锁入 MAR |
| ② | MAR→**地址总线** | 驱动地址总线，地址送到主存的地址译码器 |
| ③ | CPU→**控制总线**：MEM_Read 有效 | 通知主存"我要读" |
| ④ | 主存内部：地址译码 → 选中存储单元 → 数据送上数据缓冲 | 存储器芯片内部操作 |
| ⑤ | 主存→**数据总线**：数据有效 | 将存储单元内容输出到数据总线 |
| ⑥ | CPU 在时钟沿锁存 MDR←数据总线 | MDR 捕获数据总线上的值 |

> 地址总线在步骤②被驱动后就保持稳定直到步骤⑥完成。数据总线在步骤⑤才有有效数据——这就是为什么 MAR→ABUS 和 MEM→MDR 必须分在不同时钟周期。

### 类型二：CPU 写主存（STORE）

| 步骤 | 总线动作 | 说明 |
|---|---|---|
| ① | CPU 内部：IR(addr)→MAR | 目标地址锁入 MAR |
| ② | CPU 内部：ACC→MDR | 待写数据锁入 MDR |
| ③ | MAR→**地址总线** | 驱动地址总线 |
| ④ | MDR→**数据总线** | 驱动数据总线 |
| ⑤ | CPU→**控制总线**：MEM_Write 有效 | 通知主存"我要写" |
| ⑥ | 主存在时钟沿锁存数据 | 写入存储单元 |

> 写的顺序必须是地址先于数据稳定（步骤③在前，步骤④在后），否则主存在地址未稳定时就锁存，写到错误的单元。

### 类型三：I/O 端口读（IN 指令）

| 步骤 | 总线动作 |
|---|---|
| ① | CPU→**地址总线**：端口号（IO/M=1 标识这是 I/O 地址而非内存地址） |
| ② | CPU→**控制总线**：IOR 有效 |
| ③ | I/O 接口→**数据总线**：端口数据 |
| ④ | CPU 锁存 MDR←数据总线 |

> 和读主存几乎一样，区别仅在控制总线那一根：MEM_Read 换成 IOR（I/O Read）。统一编址的系统中（如 ARM），I/O 和内存共享地址空间，连控制信号都不区分——直接用 MEM_Read/Write，靠地址范围区分。

### 类型四：DMA 块传送（磁盘 → 主存）

| 步骤 | 总线动作 | 谁控制总线 |
|---|---|---|
| ① | CPU 设置 DMA 控制器：源地址、目标地址、字节数。此阶段 CPU 正常使用总线。 | CPU |
| ② | DMA 控制器向 CPU 发 HOLD（总线请求）。CPU 完成当前总线周期后，三态门释放总线，回发 HLDA（总线响应）。 | DMA 控制 |
| ③ | DMA 控制器驱动**地址总线**（主存目标地址），驱动**控制总线**（MEM_Write），数据从 I/O→MDR→**数据总线**→主存。 | DMA 控制 |
| ④ | 每传一个字，DMA 控制器内部地址寄存器 +1，计数寄存器 -1。若计数 ≠ 0，重复步骤③。 | DMA 控制 |
| ⑤ | 计数归零，DMA 控制器撤销 HOLD。CPU 收回总线，继续执行。结束时发中断通知 CPU。 | 交还 CPU |

> DMA 传送期间，**地址总线和控制总线由 DMA 控制器驱动，而非 CPU**。CPU 挂起（或继续执行 Cache 中的指令，不访问主存）。这就是"周期窃取"——DMA 只占用总线的存储周期，不占用 CPU 的运算部件。

### 类型五：中断响应（取中断向量）

| 步骤 | 总线动作 |
|---|---|
| ① | 外设通过**控制总线**向 CPU 发 INTR 信号 |
| ② | CPU 在当前指令末尾检测 INTR 有效且 IF=1，通过**控制总线**回发 INTA（中断响应） |
| ③ | 外设收到 INTA 后，将其中断类型号送上**数据总线** |
| ④ | CPU 锁存 MDR←数据总线，得到中断类型号 |
| ⑤ | CPU 用类型号查中断向量表（通过**地址总线**+MEM_Read），获得 ISR 入口地址 |
| ⑥ | CPU 内部：入口地址→PC |

> 中断响应过程最特殊之处在于**步骤③**——数据总线上的数据由外设驱动（不是主存也不是 CPU），这是唯一一种外设主动往数据总线上放数据的场景。外设的 INTR 和 CPU 的 INTA 是一对握手信号，走控制总线。

## 总结：五种传输的差异对比

| 传输类型 | 地址总线驱动者 | 数据总线驱动者 | 关键控制信号 |
|---|---|---|---|
| 读主存 | CPU(MAR) | 主存 | MEM_Read |
| 写主存 | CPU(MAR) | CPU(MDR) | MEM_Write |
| I/O 读 | CPU(MAR) | I/O 接口 | IOR |
| DMA 传送 | DMA 控制器 | I/O 接口（写主存） | HOLD/HLDA + MEM_Write |
| 中断响应 | CPU（取向量时） | **外设**（送类型号） | INTR/INTA |

---

## 易错点

1. **地址总线单向≠地址不能进 CPU**：地址总线是从 CPU 单向**输出**的，主存不能往地址总线上放地址。但 DMA 控制器接管后，地址总线的驱动者变成 DMA 控制器——仍是单向的，只是驱动源变了。

2. **数据总线的"双向"是分时的**：同一时刻数据总线只能被一个设备驱动。读操作时主存驱动，写操作时 CPU(MDR) 驱动，全双工不存在的，是半双工分时复用。

3. **控制总线不是一根线**：它是一组线的统称，每根线有固定的方向。MEM_Read 只能 CPU 发出，INTR 只能外设发出，不能混用方向。

4. **内部总线和系统总线是两套独立体系**：内部总线在 CPU 芯片内部（连接寄存器），系统总线在芯片外部（连接 CPU、主存、I/O）。速度差距巨大——内部总线宽度和频率远高于系统总线。
